功能

  • gomonitor stat显示采用gomonitor的taskmonitor/agsTask/agsMbyM/agsTaskmonitor4种模式投递到argo/k8s的所有项目状态汇总
  • gomonitor stat -p Id显示特定项目的详细状态，主要包括 project内各模块的依赖关系、project内各模块的运行状态、项目workflow的argoID、项目的起始时间、结束时间以及项目信息的sqlite3数据库
  • gomonitor stat -p Id -m 2，（待加入）显示项目内所有job的依赖关系及完成信息（绿色表示正在运行，蓝色表示完成，红色表示出错，默认颜色表示还未投递）

Arguments

以下为某一个账号执行gomonitor stat的示例

列名 释义
ID 项目在gomonitor sqlite3数据库中的Id
PrjName 项目名称，一般为子项目编号
Type 项目类型，例如QC，Filter，Transcriptome
Batch 项目批次
Unsubmit 项目中还未投递的job的数目，以job config ini做计数
Pending 项目中正在Pending的job数目，以job config ini做计数
Running 项目中正在Running的job数目，以job config ini做计数
Failed 项目中失败的job数目，以job config ini做计数
Succeeded 项目中成功运行结束的job数目，以job config ini做计数
Total 项目中总job数目，以job config ini做计数
Status 项目总体状态，主要包含Unsubmit、Pending、Running、Failed、Finished

查询ID为1项目的详细状态示例命令gomonitor stat -p 1

显示的信息主要包括4部分：

  1. Project内各模块的依赖关系，每一行包括2部分，[]括号内的部分是括号外模块所依赖的模块，[]内的模块以”,”相隔
  2. 如果是agsTask或者agsMbyM模式投递的project，会有AgsId，可用ags get命令查询
  3. 项目的起始以及结束时间
  4. 项目内各个模块之间的运行状态汇总

显示项目内各job的运行情况可用此命令：gomonitor stat -p 1 -m 3
