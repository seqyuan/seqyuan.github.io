功能

gomonitor agsTask <arguments> 投递的项目有以下feature

  • agsTask的主要功能是把一个project内的所有任务整体作为一个workflow通过argo软件投递到kubernetes
  • 执行投递命令会返回一个argoID，通过ags get argoID能够获得项目workflow的状态信息
  • 通过此模式投递的project能够通过gomonitor stat命令查询项目的状态，获得项目所在gomonitor数据库的Id号
  • 通过此模式投递的project能够通过gomonitor stat -p Id命令查询特定项目的详细状态，主要包括 project内各模块的依赖关系、project内各模块的运行状态、项目workflow的argoID、项目的起始时间、结束时间以及项目信息的sqlite3数据库

Arguments，gomonitor agsTask各参数说明

-i

A text file，-i参数是一个两列的文本文件，tab隔开，用于描述project中各任务之间的运行逻辑

第一列为第二列运行的前提条件，第二列可以为空。

pod config example

1-0-cut-raw.ini

单个任务的配置文件格式说明，同样适用于gomonitor submit命令

  • templetes
    1. pod_name: job的名称，请保持其在-i中的唯一性
    2. module: job所在的模块名称，一个模块可能包含多个job
    3. template: 为了减少投递到argo时配置文件的大小，设置此项，例如比对步骤不同样品采用相同的template名称
    4. imagePullSecrets: docker镜像仓库的key，找镜像仓库管理员获得，注意义乌与北京仓库此项应为不同值
  • requests
    1. memory: job需要的内存大小
    2. cpu: job需要的cpu大小
  • limits
    1. memory: job需要的内存限制，超出此限制会被k8s系统删除
    2. cpu: job需要的cpu限制，超出此限制会被k8s系统删除
  • container
    1. image: job启动容器需要的的镜像地址
    2. command: 此项为固定设置值[sh ,-c]
    3. args: 此项可以是1个脚本的绝对路径，表示此job要执行的脚本
    4. lines: 此项为数字整型，此项可设置为空，表示args设置的脚本整体按内容的顺序执行；设置其他非0整型N，则args设置的脚本会按照每N行1个单位，并行的去执行其中内容，详见gomonitor submit中的example
  • volumeMounts，此项表示执行此job要挂载的路径，此section内的key与value不固定，可以有多个key value，表示挂载多个目录。key: This must match the Name of a Volume，value用两个”|”分割成3项
    1. value的第1项：Volume’s name, Must be a DNS_LABEL
    2. value的第2项：Server, server is the hostname or IP address of the NFS server
    3. value的第3项：Path, path that is exported by the NFS server.
  • NodeSelector，此项表示job投递需要的计算资源label信息，可以设置为空，或者多个键值对，推荐设置1个键值对或者为空，为空时表示使用阿里云弹性资源，Selector which must match a node’s labels for the pod to be scheduled on that node。可选项可联系集群管理员确定，目前可选项

    -l

    Last module, from the pod templetes config section module option, if the module in the -i depend list, the pod in the module would exec at last no mater which pod its depend

-l参数可以不设置，或者指定为-i参数中各配置文件的某个module，用于表示这个module内的job在执行完成其他module之后再执行。一般情况下我们会把生产结题报告的report module放到最后去执行，report module中的各job可以保持DAG关系。

example，使用示例

以下为使用gomonitor agsTask的示例命令

gomonitor agsTask -i /abspath/all.txt -p PM-BJ181119-15-test -t FILTER

执行以上命令后会返回project投递到argo的AgsID
AgsID: pm-bj181119-15-test3-filterxf7sg
可使用ags get AgsID命令查看workflow的状态，也可以通过ags get AgsID --metrics命令查看workflow的资源消耗情况
